This article details the author's experience of overhauling the user interface and integrating new backend functionalities for the **Groovy Web Console**, an online playground for Apache Groovy users, which supports various Groovy versions (from 3 to experimental Groovy 6) and integrates with the Spock testing framework. The core narrative showcases how the author, who identifies as a backend developer lacking frontend design expertise, leveraged AI-powered tools from Google to achieve significant improvements.

The project began with a collaboration between the author and a friend, Leonard, who was working on an upcoming GitHub integration for saving and publishing scripts. This discussion sparked the idea of a comprehensive UI overhaul. Acknowledging their own limitations in frontend design, the author decided to employ AI assistance.

The first tool utilized was **Google Stitch**, an AI-powered UI/UX design tool developed by Google Labs. Stitch acts as an AI-native software design canvas, capable of generating high-fidelity, interactive user interfaces and frontend code from natural language prompts, sketches, or existing images. The author provided Stitch with a screenshot of the original Groovy Web Console interface and a clear prompt: to design a new UI resembling a code editor like Visual Studio Code, while ensuring all existing functionalities were preserved. After several iterations and minor adjustments, Stitch produced a design featuring a VSCode-like interface, complete with generated `DESIGN.md` design systems that defined typography, color palettes, and button styles. `DESIGN.md` is highlighted as Google's new open format for AI coding tools to build consistent UIs. Once satisfied, the author exported the project from Stitch as a zip file, with other export options like Figma, AI Studio, and MCP also available.

The next critical phase involved integrating this newly designed UI into the existing project, which was facilitated by the **Antigravity CLI**. Described as "The terminal-first surface to interact with Antigravity agents," this command-line interface is designed to help developers stay in their workflow without constant context switching. The author informed Antigravity CLI about the project and the intent to integrate the Stitch-generated design. Antigravity CLI guided the author through various steps to fine-tune the design's integration, adjust its behavior, update Cypress tests, and merge with upstream Git branches. The author likens the experience to working with a productive and knowledgeable co-worker. The article presents screenshots of the console's new light and dark themes, noting that while not entirely final, the progress was significant and satisfactory.

Beyond UI integration, the article also elaborates on Antigravity CLI's prior assistance with backend and infrastructure challenges related to the GitHub integration. Specifically, the team aimed to allow users to sign up with GitHub accounts and save Groovy scripts as private or public Gists. This required addressing DNS and OAuth issues, including an outdated Cloud Function (v1 instead of the required v2) used as a callback. Antigravity CLI proved instrumental here, connecting to the author's Google Cloud project, inspecting resources, making necessary configuration changes, and handling Cloud Function re-deployments. It also helped configure DNS records for the domain name provider and checked their status. Furthermore, Antigravity CLI served as a helpful companion, reminding the author of project-specific commands and even generating a Mermaid diagram to visualize the architecture of the Google Cloud project after analyzing its resources.

In conclusion, the author emphasizes the productivity of combining Google Stitch and Antigravity CLI. For a backend engineer lacking frontend expertise, Stitch enabled easy design and iteration on a modern UI, while Antigravity CLI seamlessly integrated that design and managed complex underlying tasks, including updating tests and handling cloud infrastructure. This dual approach allowed the author to achieve results that would have been unattainable otherwise, streamlining both design and development workflows. The article also points to additional resources for readers interested in learning more about Antigravity CLI.
