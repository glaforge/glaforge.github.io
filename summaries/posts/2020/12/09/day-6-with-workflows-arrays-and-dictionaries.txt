This article, part of a series on Google Cloud Workflows, introduces the use of more complex data structures – specifically arrays and dictionaries – moving beyond the simple data types (strings, numbers, and booleans) covered in previous installments. The main objective is to demonstrate how to define, manipulate, and access elements within these structures to enable more sophisticated workflow logic.

**Arrays:**
The article first details arrays, which are ordered collections of items. It explains two common ways to define them in Cloud Workflows YAML:
1.  **Inline definition:** A concise, single-line format, exemplified by `anArray: ["a", "b", "c"]`.
2.  **Multi-line definition:** A more readable format, especially for longer arrays, shown as `anotherArray: [one, two]`.

Accessing elements within an array is demonstrated using zero-based indexing (e.g., `anArray[0]` to get the first element). A provided YAML example illustrates assigning both `anArray` and `anotherArray` and then concatenating specific elements (`anArray[0]` and `anotherArray[1]`) in an output step, which results in the string `"atwo"`.

**Dictionaries:**
Following arrays, the article introduces dictionaries (also known as maps or objects), which store data as unordered collections of key-value pairs. It provides a clear YAML example of a `person` dictionary with various keys and corresponding values:
*   `firstname: "Guillaume"`
*   `lastname: "Laforge"`
*   `age: 43`
*   `kids: ["Marion", "Erine"]`

Accessing values within a dictionary is achieved using dot notation (e.g., `person.firstname`).

**Nesting and Combined Access:**
A crucial aspect highlighted is the ability to nest these data structures within each other. The `person` dictionary example explicitly showcases this by including an array (`kids`) as one of its values. The article emphasizes that this nesting capability allows for the creation of intricate data hierarchies, mirroring typical JSON or YAML structures where dictionaries can contain arrays, which in turn can contain other dictionaries, and so on.

The power of this nesting is demonstrated by showing how to access deeply nested elements. For instance, to retrieve the second child's name from the `person` dictionary, one would combine dot notation for the dictionary field with square bracket notation for the array index: `person.kids[1]`. The example confirms that concatenating `person.firstname` and `person.kids[1]` in an output step yields the string `"Guillaume and Erine"`.
