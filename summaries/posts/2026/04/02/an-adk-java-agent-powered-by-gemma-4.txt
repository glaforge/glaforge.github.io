This article provides a comprehensive guide for integrating DeepMind's newly released and powerful Gemma 4 model with ADK Java agents, offering developers multiple pathways to configure and leverage this open-weight language model. The author, a contributor to ADK Java, explores three distinct methods for connecting ADK Java agents to Gemma 4, highlighting flexibility and practical implementation steps.

**1. The Easy Case: Gemma 4 on AI Studio**
The first and most straightforward method involves calling Gemma 4 through the Google AI Studio API surface. In this scenario, Gemma 4 is exposed and treated in the same manner as other Gemini models. Developers using ADK Java would configure their agents by utilizing the `Gemini.builder()` and explicitly referencing the model name, such as `"gemma-4-31b-it"`. The article provides a Java code snippet demonstrating this, where an `LlmAgent` is built with a `Gemini` instance representing Gemma 4. It also notes an upcoming ADK release feature that will simplify this further, allowing direct model string assignment within the `LlmAgent.builder()` for even more concise configuration.

**2. Calling a vLLM Hosted Gemma 4 via LangChain4j**
The second, more advanced path, addresses integrating Gemma 4 when it is hosted via a vLLM instance, specifically in environments like Google Cloud Run with GPU support. The key distinction here is that vLLM exposes an OpenAI-compatible API, not the native Gemini API surface. To bridge this, the integration relies on the LangChain4j bridge developed by the author, which enables ADK Java to connect to third-party language models.

*   **Configuring the Chat Model:** Developers must configure an `OpenAiChatModel` (or its streaming variant, `OpenAiStreamingChatModel`) from LangChain4j. This involves setting the `modelName` (e.g., `"gg-hf-gg/gemma-4-31b-it"`), a `baseUrl` pointing to the vLLM instance's `/v1` endpoint, and optionally an `apiKey` (which might be a dummy key if not required by the vLLM setup). A significant timeout duration is advised due to potential cold start latencies when loading model weights on services like Cloud Run.
*   **Enabling Function Calling/Tool Use:** A critical configuration point for ensuring correct function calling (tool use) with Gemma 4 on vLLM is the inclusion of `customParameters`. Specifically, `chat_template_kwargs` with `enable_thinking` set to `true` is mandatory. This parameter simultaneously enables both thinking capabilities and the ability for the model to invoke external tools.
*   **Creating an ADK Agent:** The configured `OpenAiChatModel` is then wrapped using `LangChain4j.builder()`, which adapts the OpenAI-compatible model to conform to ADK's `BaseLlm` parent class, making it compatible with ADK's `LlmAgent`. An example of a "science-teacher" agent demonstrates this basic setup with a defined instruction.
*   **Adding Tools (Local Java Functions):** The article emphasizes Gemma 4's enhanced reasoning capabilities when combined with tools. ADK's `FunctionTool` allows developers to expose any local Java method as a tool. An example `orderAgent` is provided, which is instructed to use a `lookup_order` tool. This tool corresponds to a Java method (`retrieveOrder`) annotated with `@Annotations.Schema`, enabling Gemma 4 to call this function to retrieve order details based on user queries.

**3. Calling Gemma 4 Locally via Ollama**
The third integration path focuses on running Gemma 4 locally using Ollama, which provides official support for Gemma 4 models. Similar to the vLLM approach, this method also leverages the LangChain4j bridge for seamless integration with ADK Java. Developers need to configure an `OllamaChatModel`, specifying the `modelName` (e.g., `"gemma4:e4b"`) and the `baseUrl` pointing to the local Ollama server, typically `http://127.0.0.1:11434`. This offers a convenient way to experiment with Gemma 4 on a local machine.

**Conclusion**
The article concludes by asserting that the combination of ADK Java and Gemma 4 provides a powerful, flexible, and open-weight foundation for developing advanced AI agent projects. It underscores the value of the LangChain4j/ADK bridge, which significantly expands the integration possibilities by allowing Gemma 4 to be invoked through various API surfaces beyond just Google's native Gemini API. The author also includes reminders about the recent ADK Java 1.0 announcement and provides links to additional resources, including a YouTube video showcasing new features and an article detailing the creation of a specific ADK agent called "Comic Trip."
